Improved Self-Pruning for Broadcasting in Ad Hoc Wireless Networks

Proposing the ISB algorithm — a reactive broadcasting technique that uses extended three-hop neighbor knowledge to dramatically reduce forwarding nodes while keeping packet headers minimal, outperforming both self-pruning and dominant pruning across all network conditions.

📅 February 21, 2017 ⏱ 10 min read 🎓 BSc Thesis, AUST
Authors

Abstract

Reducing the number of forwarding nodes is the primary goal of any broadcasting algorithm designed for ad-hoc wireless networks. All reliable broadcasting techniques broadly fall under two categories: proactive approaches (where a node selects forwarding nodes and announces the forwarding list in the packet header) and reactive approaches (where each node cognitively determines by itself whether to forward the packet based on neighbor information).

Dominant pruning shows better performance than self-pruning by using extended two-hop neighbor knowledge. However, appending a forwarding node list to the packet header increases message overhead and consumes more bandwidth — making the approach non-scalable in large networks.

In this paper, we propose a reactive broadcasting technique — the Improved Self-Pruning based Broadcasting (ISB) algorithm — that completes the broadcast with a smaller packet header (i.e., no overhead) while using extended neighbor knowledge. Simulation results show that ISB outperforms both dominant pruning and self-pruning, with performance advantages compounding as the network grows denser and wider in transmission range.

Background: The Broadcasting Problem

A wireless ad hoc network (MANET) is a collection of two or more devices forming self-generated mobile networks with no pre-existing infrastructure — each node acts as its own router, communicating peer-to-peer. The network topology is never static; node mobility is high and available bandwidth is limited.

Simple flooding — where every node rebroadcasts to all its neighbors — ensures full packet delivery but causes severe redundancy, contention, and the broadcast storm problem. In practice, only a fraction of nodes receive the broadcast under flooding due to collisions. Intelligently selecting intermediate forwarding nodes can significantly reduce this problem.

The key challenge: reduce the number of forwarding nodes (and thus bandwidth consumption and energy expenditure) without sacrificing broadcast coverage reliability.

Self-Pruning (Reactive, 1-hop)

In self-pruning, a node J receiving a packet from sender I will forward only if the set N(J) − N(I) is non-empty — i.e., only if J has neighbours that sender I could not already have reached. This uses only one-hop neighbour information, requires no packet header modification, but misses redundancy that could be detected with wider neighbourhood knowledge.

Dominant Pruning (Proactive, 2-hop)

Dominant pruning uses two-hop neighbourhood information, allowing nodes to make more informed forwarding decisions and prune more aggressively. However, to enable this, each node must append its forwarding list to the packet header — increasing packet size with the network, making it non-scalable in large, dense deployments.

The ISB Algorithm

ISB combines the best of both: it uses extended (3-hop equivalent) neighbour knowledge for forwarding decisions, like dominant pruning — but remains reactive with no header overhead, like self-pruning.

The core idea: when node v receives a broadcast packet from node u, it constructs a set U(u,v) representing the neighbours it alone can cover. It first removes N(u) from N(v) (already covered by sender), then removes the neighbours of all nodes in N(u) that have more neighbours than v (those more connected neighbours can cover them better). If U(u,v) is empty, v drops the packet. Otherwise, v rebroadcasts.

Algorithm 1 — Improved Self-Pruning Based Broadcast (ISB)
Precondition: Node v receives an unseen packet m from u Require: v ∈ N(u) 1: U(u,v) = N(v) − N(u) 2: for each x ∈ N(u) do 3: if |N(x)| > |N(v)| then 4: U(u,v) = U(u,v) − N(x) 5: each if |N(x)| = |N(v)| and id(x) < id(v) then 6: U(u,v) = U(u,v) − N(x) 7: end if 8: end for 9: if U(u,v) = ∅ then 10: Drop the packet. 11: else 12: Rebroadcast the packet. 13: end if

The tie-breaking policy in Line 5 ensures determinism when two nodes have equal numbers of neighbours: the node with the lower ID takes responsibility. This eliminates duplicate forwarding decisions without requiring any coordination between nodes.

Simulation Results

We implemented ISB alongside self-pruning and dominant pruning in Java and ran simulations across randomly deployed networks of 100 to 500 nodes in a fixed 625 m × 625 m area. Transmission range varied from 125 m to 225 m. The metric used is the number of forwarding nodes — lower is better.

Sparse Networks (100 Nodes)

Even in sparse networks, ISB requires significantly fewer forwarding nodes than its competitors. At a transmission range of 125 m, ISB uses approximately 40 forwarding nodes vs 75 for self-pruning and 60 for dominant pruning. The advantage grows as range increases.

Moderately Dense Networks (350 Nodes)

As network density increases to 350 nodes, the performance gap widens substantially. Self-pruning plateaus near its maximum (~315 forwarding nodes regardless of range), while ISB continues to improve with wider transmission range — dropping to as low as ~50 forwarding nodes at 225 m.

Figure 5 — Forwarding Nodes in Moderately Dense Network (350 Nodes) vs Transmission Range

Dense Networks (500 Nodes)

In dense networks, the advantage of ISB is most dramatic. Self-pruning becomes nearly useless (~460 forwarding nodes), dominant pruning improves significantly with range but still requires 200–350 forwarders, while ISB maintains the lowest count of 50–130 across all tested ranges.

Figure 6 — Forwarding Nodes in Dense Network (500 Nodes) vs Transmission Range

Effect of Node Density (Fixed 150 m Range)

Holding transmission range constant at 150 m, the number of forwarding nodes grows linearly for self-pruning and dominant pruning as the network scales from 100 to 500 nodes. For ISB, growth is almost negligible — confirming that ISB's efficiency scales with network complexity rather than degrading.

Key Findings

  • ISB consistently outperforms both self-pruning and dominant pruning across all network conditions — sparse, moderate, and dense.
  • Performance advantage grows with density: as networks get denser, ISB's extended neighbour knowledge becomes more effective at identifying truly redundant forwarders.
  • Performance advantage grows with range: larger transmission ranges give more nodes the opportunity to detect redundancy — ISB exploits this better than its competitors.
  • Zero overhead: unlike dominant pruning, ISB requires no forwarding list in the packet header, keeping the approach reactive and scalable.
  • In the 12-node example topology, self-pruning, dominant pruning, and ISB require 11, 7, and 6 forwarding nodes respectively — ISB is within one of the theoretical optimum of 5.
The performance of ISB increases as the network becomes denser and the transmission range widens — precisely the conditions where scalable broadcasting matters most.

Conclusions

Broadcasting is a fundamental problem in wireless ad hoc networks. This paper proposed ISB — an improved self-pruning broadcasting technique that incurs minimal redundancy by reducing forwarding nodes significantly while maintaining zero packet header overhead.

The technique is effective for route discovery, periodic alarm signalling, data dissemination, and clock synchronisation in ad hoc networks. Future work will focus on reducing the neighbourhood information requirements further, potentially extending ISB to heterogeneous networks where nodes have different transmission ranges and battery capacities.
